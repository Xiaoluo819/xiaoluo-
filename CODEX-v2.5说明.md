# 微信批量扫码加群 v2.5 Codex 版

## 这个版本是什么

这是在原 v2.0/v2.1 项目基础上整理出的 v2.5 Codex 改进版。当前版本保留批量扫码加群主流程，清理了已废弃的导入功能残留，让入口和目录更清晰。

## 新增功能

- 校准按钮时会先截图预览点击位置，不会一开始就盲点。
- 校准截图会用红圈标出当前点，用灰圈标出最近试过的位置。
- 校准时可以先按 `p` 只预览，按 Enter 直接点击确认进入下一步；需要复查时按 `t` 测试点击。
- 测试点击时会自动生成“点击前截图”和“点击后截图”，方便判断手机有没有反应。
- 点击“加入群聊”后，会读取当前手机界面文字，判断结果。
- 如果发现“二维码已过期、无法加入、群聊人数已满、操作频繁、请稍后再试”等提示，会判定失败。
- 如果点完后还停留在“加入群聊”页面，会判定为可疑。
- 失败或可疑时会自动截图。
- 每次运行会生成 CSV 和 Markdown 报告。
- 预览模式也会生成报告，方便先看哪些二维码能识别。
- 项目内置 macOS ADB：`tools/platform-tools/adb`，新电脑通常不需要再运行 `brew install android-platform-tools`。
- 每个点位确认后会自动删除本步骤临时定位截图。
- 已清理废弃导入功能的命令、菜单入口和目录配置。

## 报告位置

运行后查看：

```text
logs/reports/
```

失败或可疑截图查看：

```text
logs/screenshots/
```

## 状态说明

- `success`：判断为成功。
- `failed`：明确失败。
- `suspicious`：结果可疑，需要看截图确认。
- `invalid`：图片无法识别二维码。
- `decoded`：预览模式下二维码可识别，但没有操作手机。

## 推荐使用顺序

```bash
bash 启动.sh
```

然后按菜单操作：

1. 连接设备。
2. 检查环境。
3. 换手机或点位不准时校准按钮。
4. 把二维码图片放入 `data/qr_input/`。
5. 先跑预览模式。
6. 确认没问题后再跑批量扫码。
7. 跑完查看 `logs/reports/` 报告。

## 校准按钮新操作

校准每个按钮时，程序会先把当前手机界面截图并打开。

- 红圈：当前准备点击的位置。
- 灰圈：最近测试过的位置。
- `p`：重新截图预览当前位置，不点击手机。
- Enter：直接点击并确认当前位置，进入下一步。
- `t`：测试点击，并打开点击前/点击后截图。
- `a`：打开上一次点击后截图。
- 方向键：小幅移动坐标。
- Shift + 方向键：大幅移动坐标。
- `+/-`：调整每次移动的步长。
- 输入 `x y`：直接跳到指定坐标。
- `y`：确认保存当前坐标。
- `q`：放弃本步骤调整，保留初始点。

## 注意

这个版本仍然基于屏幕坐标点击。微信界面、手机分辨率、弹窗、网络延迟都会影响结果。正式跑之前建议先用少量图片测试。
