## 一、项目结构

```
v2.5-codex/
├── 启动.sh            ← 双击或 bash 启动.sh 运行
├── 启动.command       ← macOS 终端双击版
├── wechat_batch.py    ← 主程序

├── requirements.txt   ← Python 依赖

│   └── adb.py         ← ADB 后端（USB + 无线）

```

## 二、环境准备（新电脑一次性操作）

### 1. 安装 ADB

ADB 用来让电脑连接并控制 Android 手机。本项目已内置 macOS 版 ADB：

```text
tools/platform-tools/adb
```

正常情况下不需要再运行 `brew install android-platform-tools`。如果软件启动时能检测到内置 ADB，会自动优先使用它。

如果另一台电脑仍然提示找不到 ADB，再按下面方式安装。

**方式 A：电脑已安装 Homebrew**

```bash
brew install android-platform-tools
```

**方式 B：电脑没有 Homebrew，手动安装**

1. 打开 Android 官方 Platform Tools 页面：
   `https://developer.android.com/tools/releases/platform-tools`
2. 下载 macOS 版本。
3. 解压后得到 `platform-tools` 文件夹。
4. 在终端进入这个文件夹：

```bash
cd 解压后的/platform-tools
./adb version
```

能看到版本号就说明 ADB 可用。后续如果软件仍提示找不到 `adb`，可以把 `platform-tools` 文件夹放到固定位置，或把里面的 `adb` 加入系统 PATH。

### 2. 进入项目目录，安装 Python 依赖

```bash
cd ~/v2.5-codex
pip3 install -r requirements.txt
```

已安装的包：`opencv-python`、`pyyaml`。

### 3. 手机端设置

**打开无线调试：**
- 设置 → 关于手机 → 连续点「版本号」7 次 → 开启开发者模式
- 设置 → 开发者选项 → 打开「无线调试」

## 三、连接手机

```bash
cd ~/v2.5-codex
bash 启动.sh
```

选 `1. 连接设备`，然后选择连接方式。

### 方式一：USB 连接（华为/荣耀推荐 🏆）

> **适用**：华为、荣耀，以及所有 Android 手机。USB 连接最稳定，不受 WiFi 影响。

**手机端准备（一次性）：**
1. 设置 → 关于手机 → 连续点「版本号」7次 → 开启开发者模式
2. 设置 → 开发者选项 → 打开「USB 调试」
3. 华为手机额外建议打开：「仅充电模式下允许 ADB 调试」

**连接步骤：**
1. 用 USB 数据线连接手机和电脑
2. 手机弹出「允许 USB 调试」→ 勾选「一律允许」→ 点确定
3. 选 `2. Android USB 连接` → 自动检测设备 → 连接成功

> ⚠️ 注意：USB 线必须支持数据传输（部分充电线不支持），建议用原装线。

### 方式二：无线连接

> **适用**：支持「无线调试」的 Android 手机（Android 11+，部分华为/荣耀机型不支持）

**已配对过的手机**：直接输入连接 IP:端口即可。

**新手机/未配对**：先连接失败 → 输入配对码 → 输入连接端口。

每次 WiFi 断开重连后端口会变，需要重新输入连接端口（配对信息不会丢）。

## 四、校准按钮（新手机/换手机必做）

选 `3. 校准按钮 + 导出坐标`。

### 流程（每步都是你先到对应画面，插件再帮你确认位置）

| 步骤 | 你需要做的 | 插件做什么 |
|------|-----------|-----------|
| 1/5 + | 打开微信到**聊天列表主页** | 先截图标出 + 的估算位置，你确认后再测试点击 |
| 2/5 扫一扫 | 手动点 + 打开菜单 | 先截图标出扫一扫位置，你确认后再测试点击 |
| 3/5 相册 | 手动进入扫一扫界面 | 先截图标出相册位置，你确认后再测试点击 |
| 4/5 选图 | 手动进入相册界面，按 Enter | **自动截图分析分割线**，定位第一行第二列 |
| 5/5 加入群聊 | 手动选一张群二维码，看到按钮后按 Enter | 先截图标出加入群聊按钮位置，你确认后再测试点击 |

### 操作方式

每一步都会先打开一张手机截图，红圈就是当前准备点击的位置，灰圈是最近测试过的位置。

每步显示类似：
```
  目标: 📷 扫一扫 → (864, 624)
  先打开截图给你看，不会立即点击手机。
```

### 截图校准模式（v2.5 Codex）

```
  📸 预览已打开: debug_before_tap.png（红圈=当前点，灰圈=历史点）
  ────────────────────────────────────
  p  预览当前位置（只截图，不点击）
  t/Enter  测试点击，并打开点击前/点击后截图
  a  打开上一次点击后截图
  ↑↓←→  微调坐标        ⇧+方向键  大幅微调
  +/-  增减步长          直接输 x y  跳转坐标
  y  确认保存当前位置    q  放弃并保留初始点
  ────────────────────────────────────
  📍 (854, 610)  偏移: (-10, -14) 步长:10px
```

| 按键 | 作用 |
|------|------|
| `p` | 重新截图预览当前位置，不点击手机 |
| `t` / `Enter` | 测试点击，并打开点击前截图和点击后截图 |
| `a` | 打开上一次点击后截图 |
| `↑↓←→` | 移动当前位置（默认 10px/次） |
| `Shift + ↑↓←→` | 快速移动（30px/次） |
| `+` / `-` | 增大 / 减小步长（1~80px） |
| `x y` | 直接输入坐标跳转 |
| `y` | 确认当前位置，保存并继续 |
| `q` / `Esc` | 放弃当前步骤调整，保留初始估算位置 |

**操作示例：**

```
进入 + 号校准
📸 先看到截图，红圈在 + 号右下角

按 ↑↑↑←←   → 目标移到 (854, 610)，显示偏移 (-10, -14)
按 p        → 只截图预览，不点击手机
按 t        → 测试点击，自动打开点击前/点击后截图

确认点对了 → 输入 y 保存 ✅
```

### 校准结果

- 保存到 `config.yaml` 的 `coords` 段
- 同时导出到 `coords_export/coords_设备型号_分辨率_时间.yaml`

## 五、批量扫码加群

### 1. 放图片

把群二维码图片（PNG/JPG/JPEG/BMP/GIF/WEBP）拖到 `data/qr_input/` 文件夹。

### 2. 预览（建议先跑）

选 `5. 预览模式`。只解码图片内容，不操作手机。看看多少张能成功解码。

### 3. 正式跑

选 `4. 批量扫码加群`。

**运行前确认：**
- ✅ 微信在聊天列表主页（哪个微信分身都行，插件不切换）
- ✅ 手机不锁屏
- ✅ 不操作手机
- ✅ 手机和 Mac 保持 ADB 连接

**运行流程：**
1. 解码所有图片
2. 逐张传到手机 → 点 + → 扫一扫 → 相册 → 选图 → 加入群聊 → 返回
3. 每次加群后等待 3-5 分钟（可配置）
4. 每 50 张暂停，按 Enter 继续

## 七、配置文件说明 (config.yaml)

```yaml
device:
  type: auto           # auto | android（ios 暂未实现）
  android_mode: auto   # auto | usb | wireless

coords:           # 按钮坐标比例（calibrate 自动生成）
  plus:           # + 号
  scan:           # 扫一扫
  album:          # 相册
  image:          # 选图位置
  join_group:     # 加入群聊按钮

rate_limit:
  min_interval: 180   # 两次加群最少间隔（秒）= 3分钟
  max_interval: 300   # 最多间隔 = 5分钟
  daily_limit:  150   # 每日上限
  batch_size:   50    # 每批数量

ui:
  join_wait: 3        # 点击加入后等待秒数

```

## 八、常见问题

### 连不上手机？

**USB 连接：**
- USB 线是否支持数据传输（不是只充电的线）
- 手机是否弹出「允许 USB 调试」弹窗未点击
- 华为手机：开发者选项 → 打开「仅充电模式下允许 ADB 调试」
- 尝试：`adb kill-server && adb start-server` 然后重连

**无线连接：**
- 同一 WiFi
- 无线调试关开一次，重新获取端口
- `adb kill-server && adb start-server` 然后重连

### 华为/荣耀手机怎么连？ 🆕

华为和部分荣耀机型没有「无线调试」选项，请使用 **USB 有线连接**：

1. USB 线连接手机和电脑
2. 开发者选项 → 打开「USB 调试」
3. 建议打开「仅充电模式下允许 ADB 调试」
4. 选 `1. 连接设备` → `2. Android USB 连接`
5. 手机弹出授权框 → 勾选「一律允许」→ 确定

USB 连接比无线更稳定，长时间批量操作推荐使用。

### 点击位置不对？
- 重新运行校准（选 3）
- 校准后坐标存在 config.yaml，下次跑自动读取

### 加群失败？
- 查看 `logs/scan_*.log` 日志
- 失败图片在 `data/qr_failed/`，选 6 可移回重试
- 常见原因：群满、链接过期、需验证

### 中途断了？
- 已处理的图片在 processed/failed 文件夹，不会重复
- 重新运行即可继续
- 断开后重连：选 1 → 直接输连接端口

### 换手机？
1. 选 1 连接新手机（选择 USB 或无线，会自动断开旧设备）
2. 选 3 校准新手机
3. 选 4 开始加群

### 想用 USB 直接跑批量？

修改 `config.yaml`：
```yaml
device:
  android_mode: usb   # 默认用 USB，每次都自动检测
```

或者命令行指定：
```bash
python3 wechat_batch.py scan --mode usb
```

### 多个微信（应用分身）？
- 插件不会强制打开微信，你打开哪个微信就用哪个
- 确保操作前微信在聊天列表主页

### 紧急停止？
- 按 `Ctrl + C`
- 已处理的不丢失

## 九、技术要点

- **坐标系统**：所有按钮坐标以屏幕宽高的比例存储，自适应不同分辨率
- **选图定位**：校准时通过截图分析相册网格的分割线，自动计算第一行第二列位置
- **微调校准**：点错后自动截图标注红圈，方向键看着手机微调，所见即所得
- **设备后端**：抽象 ADB/USB/无线/iOS 后端，USB 和无线共用同一套操作逻辑
- **ADB 通信**：通过 ADB 发送 tap 指令（USB 或无线），完全模拟手指点击
- **无侵入**：不 Hook、不注入、不修改微信，纯模拟操作
- **二维码解码**：OpenCV QRCodeDetector，支持多尺度、灰度二值化等预处理
